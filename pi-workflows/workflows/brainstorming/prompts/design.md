Turn the approved approach into a design draft.

Scale detail to complexity. Cover architecture, components/units, data flow, error handling, and testing. Emphasize isolation and clarity: each unit should have one clear purpose, well-defined interfaces, and independently testable behavior.

When working in an existing codebase, follow current patterns and include only targeted improvements that serve this goal. Avoid unrelated refactoring. Do not produce implementation steps; this design will be handed to writing-plans later.
